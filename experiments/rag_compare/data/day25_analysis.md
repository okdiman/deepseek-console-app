# День 25 — RAG-чат с памятью задачи

**Дата:** 2026-03-22
**Модель:** deepseek-chat
**Стек:** RagHook → MemoryInjectionHook → DialogueTaskHook → UserProfileHook → InvariantGuardHook
**Индекс:** 831 чанк (structure + fixed), 17 документов
**Embedding-модель:** qwen3-embedding:0.6b (локальная, Ollama)

---

## Что реализовано

Мини-чат сочетает три уровня памяти:

| Уровень | Реализация | Что хранит |
|---------|-----------|------------|
| История диалога | `ChatSession` | Все сообщения, скользящее окно |
| Рабочая память | `MemoryStore` + `MemoryInjectionHook` | Факты и долгосрочные знания о пользователе |
| Память задачи | `DialogueTask` + `DialogueTaskHook` | Goal, clarifications, constraints, explored topics |

RAG-пайплайн перед каждым ответом:
1. Обогащение короткого запроса целью диалога (если goal уже установлен)
2. Эмбеддинг запроса → Ollama
3. Поиск 10 кандидатов (pre-rerank pool) в SQLite
4. HeuristicReranker → фильтрация по порогу 0.50 → top-3
5. Оценка уверенности (confident / uncertain / weak / empty)
6. Форматирование citation block + инструкции по уровню уверенности
7. Если confident → `suppress_tools = True` (MCP-инструменты не предлагаются LLM)

Маркеры, которые агент встраивает в ответ:

| Маркер | Эффект |
|--------|--------|
| `[GOAL: ...]` | Устанавливает / обновляет цель диалога |
| `[CLARIFIED: ...]` | Фиксирует уточнение от пользователя |
| `[CONSTRAINT: ...]` | Фиксирует ограничение от пользователя |
| `[TOPIC: ...]` | Отмечает тему как разобранную; убирает совпадающий UNRESOLVED |
| `[UNRESOLVED: ...]` | Фиксирует вопрос без ответа в текущем контексте |

## Сценарий 2 — RAG System Design with Constraints

**10 сообщений.** Пользователь проектирует RAG-систему, последовательно добавляя ограничения: только Python, только локальный embedding, лёгкая БД.

### Ключевое: накопление constraints

Ограничения фиксировались немедленно при их появлении:
- Ход 2: `[CONSTRAINT: only Python, no Java or Go]`
- Ход 3: `[CONSTRAINT: embedding model must be local, no external APIs]`

В финальном ответе (ход 10) агент составил итоговую архитектуру с учётом **обоих** ограничений — без повторного напоминания, из памяти задачи.

### Цитаты и источники

- `[N]`-цитаты: **9/10 ходов (90%)**
- Среднее чанков за ход: **1.4**
- Ход 4 (выбор БД): 0 чанков из RAG — агент ответил из общих знаний, без `[N]`, но правильно порекомендовал SQLite (что совпадает с реализацией в проекте)
- Ходы про cosine similarity и chunking: RAG нашёл код из `store.py` и `chunkers.py` — цитировал реальную реализацию проекта

## Наблюдения

**Что работает хорошо:**

- Goal устанавливается на первом ходу и не теряется — даже когда пользователь меняет угол вопроса, агент удерживает общую цель
- Constraints корректно накапливаются и применяются в итоговом ответе без повторного напоминания
- UNRESOLVED работает честно: когда RAG не находит ответ — агент не галлюцинирует, фиксирует вопрос, возвращается к нему если в следующем ходу контекст появился
- Explored topics предотвращают повторение: на ход 8 («что ещё осталось разобрать?») агент перечислил только не охваченные темы

**Узкие места:**

- Avg chunks = 1.4 — порог reranker 0.50 достаточно агрессивен, часть потенциально полезных чанков отсекается
- 2/22 хода без цитат — оба случая когда RAG вернул 0 чанков; агент отвечал из общих знаний, не нарушая инструкций, но без источников
- Языковое обогащение запроса (goal + query) работает только при совпадении скриптов (кириллица/латиница); смешанные запросы пропускают этот шаг
